## Tested today

```powershell
# Property filter
pumex search --property status=published --vault sample
# → Returns only notes with `status: published` in frontmatter

# Combined tag + property
pumex search --tag wiki --property status=published --vault sample

# Property filter is fast — no FTS scan, indexed lookup
```

## Notes

Property filter queries are faster than FTS because they hit the pre-indexed properties table directly. No full-text scan needed when only filtering by frontmatter.

The `--property` flag accepts `k=v` format. Multiple `--property` flags are AND semantics.
